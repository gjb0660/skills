# Android 自动化审查与上传技能

该技能用于对待上传的提交进行结构化审查、评分，并在满足条件时通过 `repo upload` 将变更提交到内网的 Gerrit Code Review 代码仓库。

目标：在不破坏主线稳定性的前提下，自动化审查-决策-上传流程，减少人工失误并统一审查标准。

## 使用方法

步骤概览：
1. 获取待审查的 commit，优先使用上下文信息中的变更内容。
2. 按照评分准则对变更进行审查并给出 0–5 分。
3. 根据分数决定：
	- <3：拒绝并列出必须修复的问题。
	- 3–4：条件通过，自动修复并重新提交本地修改。
	- >4：审查通过，执行上传操作。
4. 执行上传（如果通过），并返回上传结果与 Gerrit 链接。
5. 用户提示词中必须明确包含 `忽略建议并强制上传`，才能绕过审查强制上传。

评分结果应包含：总体分数、主要问题摘要、建议修改以及是否允许上传。

## 代码审查准则

- 检查提交信息符合规范（模块名、描述、Change-Id等）。
- 检查代码格式符合项目规范（如缩进、命名等）。
- 检查日志中是否包含敏感信息（如密码、密钥等）。
- 检查是否残留调试代码（如 printf、log 等）。
- 确认没有明显的线程/内存/资源泄漏风险。
- 确认没有破坏现有功能的改动。

## 提交信息规范

- 提交信息的语言为英文
- 首行应包含模块名和简短描述
	- 描述以 Add/Remove/Change/Fix/Refactor/... 等短语开头
- 首行不足以描述变更时，应当添加Description分点列出具体变更内容
	- 每行的内容应简洁明了，同样以 Add/Remove/Change/Fix/Refactor/... 等短语开头
- Change-Id 由 git commit hook 自动生成并添加到提交信息末尾

示例：

```
[模块名]Add/Fix/Remove/Refactor ...

[Description]
1. Add/Fix/Remove/Refactor ...
2. ...

Change-Id: (由 git commit hook 自动生成)
```

## 上传操作

在需要上传的目录下使用以下命令：

```bash
repo upload --cbr .
```

上传成功后，输出检查点包括：Gerrit 提交 URL、变更号、以及 CI 状态（如可用）。
